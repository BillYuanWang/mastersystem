# MD Desk 教务老师使用手册

**适用版本：v0.1.23b**

**App 版本：macOS 0.1.23 build 68**

**更新日期：2026 年 7 月 22 日**

**适用学校：Starton EDU Irvine, Inc. & Master Dance**

> 这本手册写给每天实际排课、建档、报名、签到和收款的教务老师。它不讲数据库术语，按“现在要做什么”来说明该去哪里、按什么顺序操作，以及为什么有些资料不能直接删除。

**手册原则**

- macOS 的 MD Desk 只允许教务管理员帐号登录。
- App 里的正式资料会同步到学校的 Supabase 云端，不再依赖 CSV。
- 大部分保存操作会先让界面立即继续工作，再在后台同步；右下角会告诉你结果。
- 一个监护人可以管理多名少儿或成人学员，但报名永远属于具体学员，不属于监护人。
- 已经被其他资料使用的数据，通常不能硬删除。应先改正关联，或把资料停用。

## 1. 先看这里：我该去哪个 Tab

| 我现在要做什么 | 去哪里 | 最快入口 |
| --- | --- | --- |
| 看本周或未来某周的课 | 课表 | 选学期、选周、选 1 至 2 间教室 |
| 新开一门课或改错时间 | 课程 | 右上角加号，或课程行末的编辑按钮 |
| 创建家庭、学员、家长邀请码 | 家庭与学员 | 右上角加号；选中家庭后看右侧详情 |
| 给学员报课 | 报名 | 顶部始终空着的新增行 |
| 算学费、发账单、登记付款 | 账单与收据 | “新建账单”或“账单记录” |
| 给当天课程点名 | 签到 | 选日期，再选左侧课程 |
| 替家长录入或修改请假 | 请假 | 日期筛选旁边的加号 |
| 给家长发布学校消息 | 新闻 | 右上角加号 |
| 管理付费广告位 | 广告 | 右上角加号 |
| 发布家长协议、查签字 | 合同 | “协议版本”与“签署记录” |
| 建学期、假期、老师、教室等 | 数据中心 | 顶部分类切换 |

### 左侧图标看不懂怎么办

把鼠标停在左侧图标上，不用点击，就会显示该 Tab 的中文名称。最上方是 Master Dance 标志，最下方是帐号和外观设置。

### 最容易记住的一句话

**先在数据中心建基础资料，再建课程；先建家庭和学员，再做报名；有报名以后，才有签到、请假和账单。**

## 2. 第一次使用：推荐建校顺序

第一次为新学期准备资料时，建议严格按下面顺序做。这样下拉菜单不会是空的，也不容易留下半成品。

1. 打开 **数据中心**，创建学期，填写名称、开始日期、结束日期和状态。
2. 在同一个页面为该学期创建假期。即使暂时没有确定放假安排，也要先完成学校确认的假期资料。
3. 创建并整理老师、年龄段、教室、课程种类。课程种类要标明是组课还是私课。
4. 打开 **课程**，选择刚才的学期，再逐门创建课程。
5. 检查课程日期、每周时间、实际课次、定价和冲突警示。
6. 打开 **家庭与学员**，先建监护人，再在其名下建立一个或多个学员档案。
7. 打开 **报名**，把具体学员和具体课程连起来。
8. 需要收费时，打开 **账单与收据**，按报名生成费用，检查后主动发布。
9. 开课后每天使用 **签到**；请假统一在 **请假** 查看和补录。

> 不要先建一堆同名课程再回头补学期。课程复制后若还没改学期、教室或时间，系统会把冲突的两门课都标出来，提醒你立即处理。

## 3. 全局操作：登录、主题、字体、同步

### 3.1 登录

项目根目录的 `MD Desk.app` 是唯一受支持的 Mac 版本。早期只有英文课表、依赖 CSV 的原型已经停用；旧目录中的 Run 入口也会自动转到当前版本。

1. 输入教务管理员邮箱和密码。
2. 密码框右侧的小眼睛可以临时显示或隐藏密码。
3. 勾选 **记住登录** 后，最长 180 天不需要重新输入。到期后必须重新登录。
4. macOS 端不接受家长帐号。家长和成人学员帐号请使用 iPhone App。

### 3.2 邀请其他教务管理员

点击左下角帐号按钮，进入 **教务帐号**：

1. 左侧查看当前管理员。
2. 右侧填写新教务的姓名和邮箱。
3. 点击 **发送邀请**。
4. 新教务按照邀请完成密码设置后，即可使用自己的管理员帐号登录。

不要共用一个人的密码。每位教务使用自己的帐号，后续才容易查清操作来源。

### 3.3 外观模式

左下角外观菜单有三种选择：

- **跟随系统**：Mac 切换浅色或深色时，MD Desk 一起切换。
- **浅色**：无论系统外观如何，App 都保持浅色。
- **深色**：无论系统外观如何，App 都保持深色。

### 3.4 整个 App 的字体

菜单栏 **View** 中可以调整整个 App 的字体和相应排版：

- `Command +`：Font Bigger，整体字体放大。
- `Command -`：Font Smaller，整体字体缩小。
- `Command 0`：Font Default，恢复默认大小。

这和课表页右上角的“课程块字体滑块”不同。全局命令影响整个 App；课表字体滑块只影响课程色块里的文字。

### 3.5 页面状态会保留

课表看到哪一周、签到选了哪一天、请假使用了什么日期筛选、课程选了哪个学期，切到别的 Tab 再回来时都会尽量保留。需要回到今天或本周时，直接按页眉里的 **今天**、**仅今天** 或 **本周**。

### 3.6 底部学校指标

窗口最下方始终显示当前学期的：

- 活跃学员：本学期至少有一门有效报名的学员数。
- 活跃家庭：本学期至少有一名学员报名的家庭数。
- 总报名：一名学员报一门课算 1，私课也算。
- 目标：保 220，争 240。

### 3.7 同步提示怎么看

大部分新增和修改会先更新本地界面，再在后台同步云端。右下角可能出现：

- **正在同步**：可以继续做其他工作，不必反复点击保存。
- **已完成**：云端已经接受这次操作。
- **同步失败**：资料没有全部送到云端，应查看错误文字并按本手册的排查章节处理。
- **监护人码已生成**：完整邀请码只显示一次，点击查看并复制给家长。

App 处于前台并且窗口正在使用时，每 1 分钟检查一次云端变化。窗口不活跃或 App 在后台时不会持续轮询，以减少电脑和 Supabase 的占用。家庭与学员页另有手动刷新按钮，适合需要立刻查看家长刚完成的注册时使用。

## 4. 课表

课表是“看”的中心。它把课程、报名学员和当天签到情况放在同一个七天视图里。

### 4.1 选择学期和周

1. 页眉第一个菜单选择一个具体学期。课表一次只看一个学期。
2. 左右箭头切换上一周或下一周。
3. 点击中间的日期范围，会弹出日历；选任意一天，课表会跳到该天所在周。
4. 点击 **本周**，随时回到当前自然周。

切换 Tab 不会擅自把你拉回本周。

### 4.2 选择教室

点击教室选择器，可以从数据中心已有的启用教室中选择：

- 至少显示 1 间教室。
- 同时最多显示 2 间，避免课程色块被压得太窄。
- 学校未来增加教室后，不需要改程序，直接在数据中心维护即可。

### 4.3 调整密度

页眉右侧有两个滑块：

- 第一个调整时间轴比例。向右后，同样一节课会变高，细节更宽松；向左后，一屏能看到更多时间。
- 第二个只调整课程色块里的字体大小。

课程色块会根据宽度、高度和人数自动调整排版。无论缩放如何，优先保留课程名、年龄段、课程种类、老师、时间和价格。

### 4.4 课表怎么看

- 一周显示周一到周日。
- 主要时间范围按照学校常用的约 9:30 至 20:30 显示。
- 每小时是较明显的网格线，每半小时是较浅的网格线。
- 相邻日期使用轻微不同的背景明暗，帮助眼睛横向追踪。
- 不同老师使用不同课程色块颜色。
- 右上角圆圈里的 **组** 或 **私** 表示组课或私课。
- 课程色块使用实色，不会让后面的网格线穿透。

### 4.5 学员和签到颜色

课程色块会列出报名、试课和补课学员的常用姓名：

- 绿色：已出勤。
- 红色：已记录缺席。
- 灰色：还没有记录。
- 请假、试课和补课会按各自身份显示，详细情况以悬停和右侧详情为准。

把鼠标停在课程色块上，可以快速看到谁来了、谁没来、谁还未记录。点击色块后，右侧会显示完整课程资料、学员名单和签到统计，并可继续打开课程或开始签到。

### 4.6 打印

点击打印图标先进入打印预览，再确认打印。打印内容以当时选定的学期、周和教室为准。需要只打印一间教室时，先把另一间取消选择。

### 4.7 常见误区

- 课表里少了一门课：先检查课程是否属于当前学期、该周是否在课程日期范围内、该课次是否被手动移除、课程是否停用，以及教室是否被当前视图选中。
- 同一时段没看到全部教室：教室选择器最多同时显示两间，需要切换组合查看。
- 课表里不要直接改课程资料：点击右侧 **打开课程**，进入课程 Tab 编辑。

## 5. 课程

课程 Tab 管理每周重复的课程规则和由此生成的实际课次。

### 5.1 先选择查看范围

页眉可以选择某一学期，也可以选择 **全部学期**：

- 日常检查和编辑时，建议选具体学期，避免同名课程混淆。
- 查历史课程或准备复制到新学期时，使用全部学期。
- 选中的学期会保留，切走再回来不会自动改变。

### 5.2 找课程

- 使用右上角搜索框做全表搜索。
- 每列表头都可以排序或筛选。
- 表格会根据内容和窗口宽度自动分配列宽；太长的内容会用省略号显示，悬停或打开编辑可看完整内容。
- 表格包括课程名、学期、年龄段、教室、老师、时间、实际课次、价格、课程种类、冲突、状态和操作。

### 5.3 新建课程

点击右上角加号，依次填写：

1. 课程名称。
2. 学期。
3. 年龄段。
4. 教室。
5. 授课老师。
6. 课程种类。
7. 是否启用。
8. 开始周、结束周、星期和上课时间。
9. 定价状态与单价。
10. 备注。

如果学期尚未建立假期资料，编辑器会提示先去数据中心补齐。

### 5.4 实际课次和休息周

系统会根据开始周、结束周、星期和时间自动生成每一次课，并排除该学期的假期。右侧会看到一块块实际日期：

- 点击某个日期右上角的叉，可以单独移除那一周。
- 移除适合临时停课、老师请假或学校活动。
- 修改日期、星期或时间后，实际课次会重新计算。
- 保存前检查“有效课次 / 总课次”是否符合预期。

### 5.5 定价

课程定价状态有：

- **待定价**：课程可以先建立，但费用尚不能完整计算。
- **已定价**：单价已确认。
- **免费**：费用为 0。
- **需复核**：原有价格或课次变化后，需要教务重新确认。

组课可以设置：

- 整期每节价：整期报名时使用，学期费用等于单价乘实际收费课次。
- 按次每节价：只有填写后，该组课才开放按次报名。

私课只设置 **私课按次价**，并且只能选择具体课次报名，没有整期报名。

### 5.6 编辑、复制和冲突

- 点击铅笔可以编辑课程的全部资料，包括学期、日期和时间。
- 点击复制会建立一份副本，再打开编辑。复制历史课程到新学期时，第一件事应改学期和日期。
- 如果同一教室或同一老师的时间重叠，冲突的两门课程都会出现警示。警示是前端即时判断，不会悄悄替你改时间。
- 修正学期、老师、教室或时间后，警示会消失。

### 5.7 删除与停用

没有实际课次或报名的错误课程可以删除。已经生成课次或已有报名的课程不会被硬删除，应编辑后关闭 **启用课程**。停用能保留历史记录，也不会破坏签到和账单。

## 6. 家庭与学员

这一页的结构是：**一个监护人家庭，下面可以有一个或多个学员档案。** 学员可以是少儿，也可以是成人本人。

### 6.1 表格怎么看

表格每行是一个监护人家庭，列包括：

- 监护人。
- 邮箱。
- 电话。
- 学员档案，多人时用逗号隔开。
- 帐号状态：已连接或待认领。
- 总报名：自 2026 年秋季开始的累计报名数。
- 学期报名：页眉所选统计学期内的报名数；未选学期时显示短横线。

每列都有自己的筛选。顶部搜索会搜索全部家庭资料。需要清空多个列筛选时，点击清除筛选按钮。

### 6.2 新建监护人

点击右上角加号：

1. 填写监护人姓名。
2. 填写帐号邮箱，必填并检查格式。
3. 填写电话，必填。输入纯数字或常见格式后，系统会整理成 `+1 (000) 000-0000`。
4. 可填写额外联系邮箱和家庭地址。
5. 保存。

帐号邮箱将来是家长登录名。家长在 iPhone 端不能自行修改该邮箱；教务可以在 Mac 端更正。

### 6.3 添加学员档案

选中家庭后，在右侧 **学员档案** 点击加号：

- 填写常用姓名。课表和签到优先显示这个名字。
- 证件姓名、生日可以按需要填写。
- 选择少儿或成人本人。
- 确认归属的监护人。
- 可以把多个孩子、家长本人都放在同一个家庭下。

### 6.4 让家长认领帐号

家庭创建后，在右侧帐号区域点击 **生成监护人码**：

1. 完整邀请码只显示一次，立即复制并安全地发给家长。
2. iPhone 家长注册时先输入邀请码，不需要重新填写邮箱。
3. 系统显示教务预先记录的邮箱，家长不能修改。
4. 家长阅读当前合同，滚动到底并签字。
5. 家长输入两次新密码，点击 **同意并注册**。
6. 注册完成后回到登录页，再用邮箱和新密码登录。
7. Mac 端点手动刷新，帐号状态应变为 **已连接**。

邀请码过期或遗失时，可以重新生成。旧码随即失效。

### 6.5 在家庭详情里管理课程

右侧可以选择某个学员，查看已报课程：

- 已选课程可从这里移除。
- 也可以从现有课程中添加报名。
- 私课必须选择具体课次。
- 大批量报名更建议去 **报名** Tab，信息更完整、筛选更方便。

### 6.6 删除安全规则

家庭和学员只能由 macOS 教务管理员删除，iPhone 端没有删除权限。每次删除前都会警告：

- 未连接帐号、没有报名、签到、请假或签字记录的空家庭，可以永久删除。
- 已连接帐号或已有正式记录时，系统会阻止删除。
- 已有历史记录的学员应改为停用，不要破坏历史链条。

## 7. 报名

报名是 **课程 × 学员**。监护人负责帐号和家庭，但真正上课、签到和计费的对象一定是具体学员。

### 7.1 查看报名

页眉显示所选学期的：

- 总报名。
- 组课报名。
- 私课报名。

可以选择一个学期或全部学期，并按学生、家庭或课程搜索。表格会显示学员、课程、学期、时间、老师与教室、报名方式、估算学费、计费状态、报名状态和日期。

### 7.2 新增报名

表格顶部始终保留一行空白新增行，不需要每次先打开弹窗：

1. 点击学员格，从按家庭分组的学员总览中选择具体学员。
2. 点击课程格，从按星期和时间整理的微型课程列表中选课。同名课程也能靠学期、星期、时间、老师和教室区分。
3. 选择报名方式。
4. 若为按次报名，勾选具体课次。
5. 点击行末确认按钮。

提交后界面会立即给出反馈，云端同步在后台进行。

### 7.3 整期和按次

- 组课默认可整期报名。
- 组课只有在课程里填写了按次每节价，才可按次报名。
- 私课永远是按次报名，必须选择具体日期。
- 整期报名包含该课程全部未取消课次，但计费可以根据报名起始日、试课和折扣调整。

### 7.4 编辑这份报名的计费

点击报名行末的金额按钮，可以为这名学员的这门课设置：

- 报名方式和具体课次。
- 计费状态：待定价、已就绪或需复核。
- 实际使用的每节单价。它是报名价格快照，课程以后改价不会无声改掉已经确认的报名。
- 试课费。试课费独立计算，不参与课程折扣。
- 一个课程折扣，可选择百分比或固定金额，并自定义折扣名称。
- 计费开始日等报名级调整。

一门报名当前只支持一个折扣。注册费、结余和其他家庭费用不在这里填，应在账单里添加。

### 7.5 移除和同步失败

- 行末垃圾桶用于移除报名；如已有强关联记录，系统会给出限制。
- 正在同步的行会显示状态。
- 失败行可以重试，也可以取消本地这条待同步报名。
- 不要因为网络稍慢连续创建同一报名。先看行状态和右下角同步提示。

## 8. 账单与收据

这一页有 **新建账单** 和 **账单记录** 两个部分。账单是收费通知，收据是付款后凭证。

### 8.1 核心规则

- 账单以家庭为单位，但每个收费项目可以归到某个具体学员，方便家长看每个孩子花了多少钱。
- 同一家庭、同一学期只有一条账单系列。
- 已发布账单不能直接修改。需要纠正时创建新版本，旧版本仍可查看。
- 新家庭或新学期会建立新的账单系列和账单编号。
- 所有生成都由教务主动触发，不会自动给家长发账单或催款。

### 8.2 新建账单

1. 选择监护人家庭。
2. 选择学期。
3. 确认账单编号、版本、学年和出具日期。
4. 点击 **按报名生成**，从当前有效报名自动生成学费项目。
5. 按实际情况添加或调整其他项目。
6. 检查每个项目属于家庭还是具体学员。
7. 检查 **应付** 开关和本次应付总额。
8. 填写备注。
9. 查看 PNG 预览，确认后出具。

如果该家庭和学期已经有账单，系统会基于最新版本进入“创建账单新版本”，不会偷偷建立第二个根账单。

### 8.3 可添加的项目

- **按报名生成**：按每节单价、实际收费课次、报名起始、试课和报名折扣计算。
- **注册费**：由教务决定是否添加。少儿学员按秋季至次年夏季的学年收取一次，成人学员不收；系统当前不主动提醒。
- **上期结余（抵扣）**：家庭余额，使用负金额抵扣。
- **上期欠款**：家庭历史欠款。
- **其他项目**：教务自定义名称和金额。

关闭某行的 **应付** 后，该行仍展示在 PNG 上，但不计入本次应付。例如已经付款、只想展示给家长看的项目。

### 8.4 查看历史和创建新版本

在 **账单记录**：

1. 用学期筛选或搜索家庭、账单编号。
2. 每个家庭和学期显示一条系列记录。
3. 右侧版本菜单可查看 `v1`、`v2` 等历史版本。
4. 历史版本只读。
5. 点击 **新版本**，从最新版本开始纠正并重新出具。

新版本发布后，旧版本会标记为已被替代，但不会消失。

### 8.5 记录付款

只能在当前有效账单上记录付款：

1. 点击 **记录付款**。
2. 填写本次付款金额，或按 **付清**。
3. 选择现金、支票、Zelle 或银行卡。
4. 选择付款日期并填写备注。
5. 点击 **生成收据并记录**。

银行卡会在应付金额外独立加收 3.5% 手续费。部分付款可以记录，但学校原则上不建议；部分欠款按整个家庭追踪。

### 8.6 PNG 保存和发送

账单和收据 PNG 会：

- 私密保存到 Supabase，供已连接家庭在 iPhone 的 **合同与收据** 页面查看。
- 同时保存在当前 Mac 的 `~/Documents/MD Desk Docs/<家庭名称>/`。
- 可以点击复制 PNG，再直接粘贴到微信等聊天工具。
- 可以点击文件夹按钮打开 MD Desk Docs。

当前系统不计算税，不制作退款账单，也不自动催款。

## 9. 签到

签到页适合上课当天连续操作，也支持选择过去日期补录。

### 9.1 选择日期和课程

1. 点击 **今天** 回到当天。
2. 也可以用日期选择器查看过去或未来日期。
3. 左侧选择当天某一节课。
4. 右侧会显示正式报名学员，以及单独的试课和补课区域。

页眉会汇总出勤、请假、试课和补课数量。

### 9.2 正式报名学员的状态

对正式报名学员可以记录：

- 出勤。
- 缺席。
- 取消当前签到状态，恢复为未记录。

已经存在的请假会显示为家长请假或教务请假。请假的新增、修改和删除统一去 **请假** Tab，不在签到页制造另一份记录。

### 9.3 试课

试课学员没有正式报名这门课：

1. 在试课区域点击添加。
2. 从现有学员中选择。
3. 保存后，该学员只算这一次试课，不自动变成报名。
4. 后来正式报名时，这一次仍保留为试课，不改算普通出勤。

试课费在报名计费中单独填写。

### 9.4 补课

补课学员通常在另一门已报名课程中缺席或请假，再来这门没有报名的课：

1. 在补课区域点击添加。
2. 选择学员。
3. 选择对应的原缺席或请假课次。
4. 保存后，系统把本次补课和原课次联系起来。

补课可以参与家长端的全勤状态计算。请尽量选对原课次，不要只写备注代替关联。

### 9.5 撤销错误操作

出勤、缺席、试课和补课都可以取消，恢复为未记录。发现点错时直接撤销，不需要再建立一条相反记录。

### 9.6 操作建议

- 上课前先确认日期和课程名，尤其是同名课程。
- 一节课点完后看人数汇总，再切下一节。
- 网络同步在后台进行；不要因为动画或状态提示重复点击同一个学员。
- 需要快速只看结果时，也可以回到课表，把鼠标停在课程色块上。

## 10. 请假

请假只有“已请假”这一种业务结果，不需要批准、拒绝或多级状态。

### 10.1 查看和筛选

- 点击 **仅今天**，只看今天的请假。
- 日期选择器可以看指定日期。
- 点击日期旁边的清除按钮，显示全部日期。
- 表格会显示学员、课程、具体课次、来源和备注。
- 来源会区分家长 iPhone 提交或教务录入。

选择的日期筛选会保留，切换 Tab 后回来仍在原来的视图。

### 10.2 教务新增请假

点击右上角加号：

1. 选上课日期。
2. 选该日具体课次。
3. 选这门课已报名的学员。
4. 填写可选备注。
5. 保存。

教务可以在任何时间补录，包括课程开始前 12 小时内和课程结束以后。

### 10.3 编辑和删除

每条请假都可以编辑或删除。删除前会再次确认。需要撤销错误请假时直接删除，不要把备注改成“取消请假”。

### 10.4 家长端限制

家长必须在课程开始前至少 12 小时请假。少于 12 小时时，iPhone 会说明已过自助请假时间，并建议联系教务老师。家长提交后即为请假，不需要教务批准。

## 11. 新闻

新闻由 Mac 教务发布，家长 iPhone 首页显示最新三则，其余可进入查看更多。

### 11.1 新建新闻

点击右上角加号，填写：

1. 标题，必填。
2. 作者，必填。
3. 首页摘要，可不填；空白时会使用正文首行。
4. 正文，必填。使用空行分段。
5. 封面图，发布时必填，建议 1:1。
6. 正文图片，可多张，并指定放在第几段之后。

封面图只用于列表缩略图，不会在正文顶部重复。正文只显示专门上传的正文图片。

### 11.2 图片处理

- 支持 JPEG、PNG、HEIC。
- 过大的图片会自动按比例缩小并压缩，不裁切内容。
- 封面建议 1:1，iPhone 列表会完整显示。
- 图片上传和云端同步可能比纯文字慢，看到同步提示时可以继续工作，但发布前要等成功结果。

### 11.3 草稿、发布和撤回

- **保存草稿**：家长看不到，适合未完成内容。
- **发布**：符合必填条件后向家长显示。
- **保存修改**：更新已经发布的新闻。
- **撤回**：从家长端下架，资料保留，可重新发布。
- **删除**：永久删除新闻和全部图片，无法撤销。

先用撤回代替删除，除非确认这篇新闻和图片以后都不需要。

## 12. 广告

广告页管理家长 iPhone 首页的商业广告。当前共有 5 个广告位，标准价格为每月 99 美元。

### 12.1 新建广告

1. 点击右上角加号。
2. 选择广告位 `#1` 至 `#5`。
3. 填首页标题或品牌名称，最多 40 字。
4. 填广告正文，最多 1024 字。
5. 选择起始日和终止日，结束日包含在投放期内。
6. 可用 1、3、6、12 个月快捷按钮。
7. 上传方形缩略图和广告海报。
8. 保存草稿或发布。

### 12.2 广告位和计费

- 同一个广告位的日期不能和另一条广告重叠。
- 系统会在保存前显示冲突警示。
- 不足一个完整月按一个月计算。
- 页面会显示计费月数和预计费用。
- 费用只是教务管理用估算，不会自动建立家庭账单。

### 12.3 两张图片的用途

- **方形缩略图**：用于 iPhone 首页广告列表，要求 1:1，最低 600 × 600。
- **广告海报**：进入广告详情后按宽度展示，可以是任意比例。
- 两者都支持 JPEG、PNG、HEIC；超限时自动按比例优化到 7 MB 内，不裁切。

### 12.4 状态

- 草稿：家长看不到。
- 已发布但日期未到：等待投放。
- 投放中：当前日期在投放范围内。
- 已结束：终止日已过。
- 已撤下：人工停止，可重新投放。
- 永久删除：会连同两张图片删除，无法撤销。

iPhone 首页的新闻和广告是两个独立区域。广告区最下方会提示商家联系佳美教务老师咨询有限广告位。

## 13. 合同

合同是 App 内的文字协议，不是 PDF。它有版本，并保存家长的真实签字图。

### 13.1 发布协议

在 **协议版本** 点击右上角加号：

1. 选择所属学期。
2. 填协议名称。
3. 填完整正文，至少 20 个字。
4. 点击 **保存并发布**。

编辑旧协议时，实际动作是 **发布协议新版本**，不会偷偷改掉家长过去签署的原文。

### 13.2 新版本的影响

发布新版本后，所有监护人下次登录 iPhone 时都需要重新阅读、滚动到底并签名。只有完成当前版本签署后，才能继续正常使用家长端。

因此，小错别字也会触发全体重签。发布前应由学校负责人确认文本。

### 13.3 签署记录

切换到 **签署记录** 可以看：

- 签署人。
- 签字缩略图。
- 合同版本。
- 签署范围。
- 同意时间。

把鼠标停在签字缩略图上，会放大显示原签字，便于核对。

### 13.4 删除限制

没有签署记录的错误协议可以删除。已经有人签署的合同不会被删除，因为签字必须对应当时看到的原文。需要停止使用时发布新版本或停用旧版本。

## 14. 数据中心

数据中心维护全校共用的基础资料。课程编辑器里的下拉菜单来自这里，不是系统预设。

### 14.1 学期与假期

左侧建立学期：

- 学期名称。
- 开始日期。
- 结束日期。
- 草稿、开放或结束状态。

右侧建立假期：

- 所属学期。
- 假期名称。
- 开始和结束日期。
- 备注。

假期会影响实际课次。课程建立后，学期和假期通常不能直接删除，只能修正或保留历史。

### 14.2 老师、年龄段、教室和课程种类

顶部切换资料类别后，可以新增、编辑、停用或尝试删除：

- **老师**：姓名、备注、启用状态。
- **年龄段**：名称、备注、启用状态。
- **教室**：名称、启用状态。
- **课程种类**：名称、备注、启用状态，以及属于组课还是私课。

课程分类是旧兼容字段，App 里不显示也不维护。请使用课程种类。

### 14.3 排序

这些基础资料的顺序会影响创建课程时下拉菜单的顺序：

- 在手动排序模式，拖动每行前面的三条横线改变顺序。
- 点击列表头可按某列升序、降序，再次点击回到手动顺序。
- 点击表头最左边的三条横线可立即恢复手动排序。

经常使用的年龄段、老师或教室可以拖到前面，提高连续建课速度。

### 14.4 删除为什么会失败

如果某个年龄段、教室、老师、课程种类、假期或学期已被课程等资料使用，系统会阻止删除。正确做法通常是取消 **启用**，让它不再出现在新建资料的常用选择里，同时保留历史课程的解释能力。

## 15. 常见完整流程

### 15.1 新学期开课

1. 数据中心：新建学期，录入所有假期。
2. 数据中心：检查老师、年龄段、教室和课程种类，停用过时项并调整排序。
3. 课程：切到全部学期，复制上一学期可复用课程。
4. 课程：逐门改成新学期，调整日期、时间、价格和休息周。
5. 课程：清除所有教室或老师冲突警示。
6. 报名：选择新学期，为学员建立报名。
7. 账单与收据：按家庭和新学期生成账单。
8. 课表：检查第一周和假期前后周，确认实际课次正确。

### 15.2 新家庭入学

1. 家庭与学员：创建监护人，核对邮箱和电话。
2. 家庭与学员：在家庭下建立每位实际上课者的档案。
3. 家庭与学员：生成一次性监护人码，马上复制给家长。
4. 家长 iPhone：邀请码验证、读合同、签字、设密码、登录。
5. Mac：手动刷新，确认帐号从待认领变为已连接。
6. 报名：给具体学员选课，不要给监护人“报名”。
7. 账单与收据：按报名生成并出具该家庭账单。

### 15.3 试课后正式报名

1. 签到：在实际来访课程的试课区域添加学员。
2. 之后决定报名时，到报名 Tab 建立正式报名。
3. 打开该报名的计费编辑，填写试课费和正常计费起始。
4. 原试课记录保持试课，不改成普通出勤。
5. 账单按报名生成时，检查试课费和正常课次是否分开。

### 15.4 请假后补课

1. 家长 iPhone 提交请假，或教务在请假 Tab 录入。
2. 签到页会显示该课次已请假。
3. 学员到另一节未报名课程补课时，在补课区域添加。
4. 选择对应的原请假或缺席课次。
5. 如点错，可取消补课状态并重新选择。

### 15.5 发账单并收款

1. 先在报名 Tab 确认每份报名的价格、试课费和折扣。
2. 账单与收据：选家庭和学期，按报名生成。
3. 添加注册费、结余、欠款或其他项目。
4. 检查家庭总额和每个学员子项目。
5. 发布账单 PNG。
6. 收到付款后，在账单记录中选择最新有效版本。
7. 记录付款方式和金额，生成收据 PNG。
8. 需要微信发送时，点击复制 PNG。

### 15.6 发布内容

- 新闻：先存草稿，确认正文分段、封面和正文图片，再发布。
- 广告：检查广告位、日期冲突、两张图片和预计费用，再发布。
- 合同：先由负责人完整校对；一旦发布新版本，所有家长都要重新签。

## 16. 数据关联、删除与同步规则

### 16.1 资料之间怎么连

| 资料 | 依赖什么 | 后面会被什么使用 |
| --- | --- | --- |
| 学期 | 无 | 假期、课程、合同、报名统计、账单 |
| 假期 | 学期 | 课程实际课次 |
| 老师、年龄段、教室、课程种类 | 无 | 课程 |
| 课程 | 学期和基础资料 | 实际课次、报名、签到、请假、账单 |
| 监护人 | 联系资料 | 学员、帐号连接、账单、合同签字 |
| 学员 | 监护人 | 报名、签到、请假、试课、补课 |
| 报名 | 课程和学员 | 签到、请假、计费 |
| 账单版本 | 家庭和学期 | 付款、PNG 文件 |

### 16.2 删除原则

系统会尽量允许撤销，但不会为了“看起来删掉了”而破坏历史：

- 尚未被使用的输入错误资料，可以删除。
- 已被引用的基础资料，应停用或编辑名称。
- 课程已有课次或报名时，应停用。
- 家庭或学员已有帐号、报名、签到、请假或签字时，不能硬删除。
- 签到状态、试课、补课和请假可以取消或删除，因为这属于可撤销的业务操作。
- 已发布账单不可改写，只能创建新版本。
- 已签署合同不可删除原文，只能发布新版本或停用。
- 新闻和广告允许永久删除媒体，但删除不可恢复，通常先撤回。

### 16.3 哪些资料需要跨设备同步

以下资料在任何合法客户端修改后，其他已授权客户端都应看到更新：

- 学期、假期、课程、实际课次和基础资料。
- 监护人、额外联系邮箱、地址、学员档案和帐号连接状态。
- 报名及其按次课次、价格快照、试课费和折扣。
- 出勤、缺席、请假、试课、补课和撤销后的未记录状态。
- 新闻、新闻图片、广告、广告图片和投放日期。
- 合同版本、签字和同意时间。
- 账单版本、付款、账单 PNG 和收据 PNG。

本地界面会优先快速响应，云端负责保存事实和跨设备分发。看到同步失败时，不应把“本地已经显示”误认为云端已经成功。

### 16.4 隐私

- 监护人帐号只能看到自己连接的家庭。
- iPhone 家长端不能删除家庭或学员，也不能修改帐号邮箱。
- 账单、收据和签字属于私密资料，不应放进公开文件夹或 GitHub。
- 不要截图或复制完整邀请码到公共群；邀请码只能使用一次。

## 17. 常见问题排查

### 17.1 右下角显示“同步失败”

1. 先读失败提示，确认是哪一类资料。
2. 检查网络连接。
3. 不要连续重复新增同一条资料。
4. 保持 App 在前台活动窗口，等待下一次同步检查。
5. 家庭资料可点击手动刷新。
6. 如果失败一直存在，记录失败文字、操作对象和时间，再交给维护人员排查。

### 17.2 Mac 有资料，iPhone 看不到

1. 确认 Mac 右下角已经显示同步成功。
2. 确认 iPhone 使用的是正确家庭帐号。
3. 把 iPhone App 切到前台并等待同步，必要时重新打开。
4. 对家庭资料，确认监护人帐号确实显示已连接。
5. 新闻或广告还要检查是否已发布，以及广告日期是否正在投放范围内。

### 17.3 iPhone 有新资料，Mac 看不到

保持 MD Desk 为前台活动窗口，等待最多 1 分钟；家庭与学员页可以点击手动刷新。请假也可切换日期筛选后重新查看。

### 17.4 新建课程时下拉菜单为空

去数据中心检查：

- 是否已经建立学期和假期。
- 老师、年龄段、教室、课程种类是否存在并启用。
- 当前选择的学期是否正确。

### 17.5 课程保存后出现冲突

冲突通常表示同一时间使用同一教室，或同一老师同时上两门课。打开被标警示的两行，检查学期、星期、开始结束时间、教室和老师。复制课程后最容易出现这种情况。

### 17.6 删除按钮报错

这通常不是权限故障，而是资料已经被引用。按错误提示检查报名、课次、签到、请假、签字或帐号连接。确有历史数据时改为停用；不要为了删除而先删正式历史。

### 17.7 账单不能修改

已发布账单是正式记录，所以只读。到账单记录选择最新版本，点击 **新版本**。旧版本仍能通过版本菜单查看。

### 17.8 新闻或广告发布按钮不可用

- 新闻：检查标题、作者、正文和封面图。
- 广告：检查标题、正文、日期、广告位冲突、方形缩略图和海报。
- 等待图片自动优化完成；图片过大时系统会缩小和压缩，不裁切。

### 17.9 课表看不到学员或颜色不对

检查：

1. 学员是否真正报名这门课，或是否以试课、补课加入这一次。
2. 当前课表周是否包含该课次。
3. 签到记录是否属于同一天同一节课。
4. 请假是否已同步。
5. 鼠标悬停看详细名单，再和签到 Tab 对照。

### 17.10 App 字太大导致拥挤

先按 `Command 0` 恢复全局默认字体。只想调整课表课程块时，使用课表页第二个滑块，不要放大全局字体。

### 17.11 打开后只有英文课表

这表示打开的是已经停用的早期 CSV 原型，不是当前 MD Desk。退出该窗口，回到项目根目录运行 `./script/build_and_run.sh`。从 `v0.1.23b` 开始，旧 App 已删除，旧构建入口也会自动转到新版。

## 18. 状态与颜色速查

| 名称 | 意思 | 建议动作 |
| --- | --- | --- |
| 待认领 | 监护人资料已建，但尚未连接 iPhone 帐号 | 生成监护人码给家长 |
| 已连接 | 家长帐号已经连接这个家庭 | 可正常下行家庭资料 |
| 课程待定价 | 课程还没有可确认单价 | 回课程编辑补价 |
| 课程需复核 | 日期、课次或价格变化后需重新确认 | 检查课程价格和报名 |
| 报名已就绪 | 报名计费信息可用于生成账单 | 进入账单按报名生成 |
| 绿色学员名 | 已出勤 | 无需操作 |
| 红色学员名 | 已缺席 | 核对是否应请假或补课 |
| 灰色学员名 | 尚未记录 | 到签到页处理 |
| 正在同步 | 本地已操作，云端处理中 | 可以继续工作，避免重复提交 |
| 同步失败 | 云端未接受全部操作 | 查看错误并按排查章节处理 |
| 草稿 | 只有教务看得到 | 完成后发布 |
| 已撤回或已停用 | 保留历史，但不再正常展示或选择 | 需要时可重新启用或发布 |
| 账单已替代 | 已有更新版本 | 使用最新版本收款 |

### 当前版本暂不提供

- 家长自行选课。
- 独立老师帐号。
- 自动催费或费用提醒。
- 税费计算。
- 退款账单生成器。
- 多个折扣叠加。
- 可编程的特殊情况规则引擎。
- iPhone 端的课程定价控制。

这些边界是当前产品选择，不代表资料丢失或 App 故障。
